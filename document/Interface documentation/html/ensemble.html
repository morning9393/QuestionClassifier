ensemble module¶

class ensemble.QuestionClassifier(ensemble_size, data_path, vocabulary_path, labels_path, stop_words_path, pre_train_path=None)¶

Bases: object

Ensemble of based model, if ensemble size is 1, means a single model and will not apply ensemble strategy.

bootstrapping()¶

Generate a subset with randomly selecting N sample in dataset with replace. N is the size of dataset.

Returns

a subset with the same size of original dataset.

init_subsets(ensemble_size)¶

Use bootstrapping to generate a set of sub dataset, which used to build different models.

Parameters

ensemble_size – How many model in the committee. If 1, bootstrapping will not be applied.

load(path)¶

Load models form path to build this ensemble.

Parameters

path – Model or sub-models path.

load_dataset(path)¶

Load dataset from file, transfer to a list and assign to self.dataset.

Parameters

path – Data file path.

static normalize(sample)¶

Normalize a list(question) of indexes to a unified length 21. Used when model is cnn.

Parameters

sample – A list(question) of indexes.

Returns

Unified list with length 21.

save(path)¶

Save every model in ensemble to path.

Parameters

path – Path to save.

Returns

test(data_set, is_cnn=False, print_detail=False)¶

Validate this model with corresponding test set.

Parameters

  • data_set – Test set, QuestionSet type.

  • is_cnn – Is this model based on cnn or not.

  • print_detail – Print every sample and its result in the test set or not.

Returns

A tuple with hit number and accuracy rate.

train(model, embedding_dim, lstm_hidden, fc_input, fc_hidden, epochs, lr, freeze=True, test_path=None)¶

Train all the models in committee.

Parameters

  • model – Type of model, must be one of [bilstm/hybrid-cat/hybrid-add/cnn/bow].

  • embedding_dim – Length of reserved vocabulary, used to build embedding layer.

  • lstm_hidden – Dimension of lstm hidden state, used to build bilstm model.

  • fc_input – Dimension of input full connect layer.

  • fc_hidden – Dimension of hidden full connect layer.

  • epochs – How many epochs it trains.

  • lr – Learning rate.

  • freeze – Freezing embedding layer or not. if True, weight in embedding layer will not be changed during training.

  • test_path – Test file path, use to validate the model after every epoch.
